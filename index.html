Responsive is the developers framework.

Accessible

Each component and plugin within the framework has been carefully crafted to be fully accessible to keyboard and screenreaders.

Flexible

Responsive has built-in right-to-left language support for all its components, even touch events have been altered to provide the best possible experience for right-to-left users.

Extensible

Built with Sass and jQuery, the best tools in their class, Responsive has been build to cover every need for a modern website. That said it can be easily extended and enhanced to suite your specific needs.

Why Responsive?

Frameworks like Bootstrap and Foundation are too design opinionated and heavy. They're great for prototyping but every time you start a real, front-facing, project with them you have to overwrite lots of designer styles that do nothing to add to the functionality of the website. That costs developers time and money.

Responsive has been built with that in mind. It is the result of thousands of hours of real, client driven web development and testing; specifically developed to be as lightweight as possible to prevent the need to undo styles set by the framework itself and allow developers to write efficient code and lower costs.

Responsive is tiny. The combined output CSS and JavaScript is only {{site.buildsize}}kb minified and gzipped but there is a lot of functionality built into the framework with touch, right-to-left language, and accessibility support. It's designed to be dropped-in, as-is to your website such as you would with Normalize.css.
