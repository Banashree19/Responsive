Accordion

A jQuery plugin that creates either grouped or ungrouped accordions for toggling the visibility of content. Uses CSS transitions to perform the animation.

Comes with a markup API that allows running the plugin without writing a single line of JavaScript. This is Responsive's first class API and should be your first consideration when using a plugin.

The Markup

<div class="accordion">
    <div class="accordion-head">
        <a data-accordion-target="#cllps1" class="accordion-toggle" href="#">
            Accordion Header Trigger                               
        </a>
    </div>
    <div class="accordion-body expand" id="cllps1">
        <p>
            Lorem ipsum dolor sit amet...                              
        </p>
    </div>
</div>

Methods

The accordion exposes the following method signatures.

.accordion(options)
Initialises the plugin with an optional options object
options
Contains parameters to pass to the plugin.
target
A jQuery selector indicating what target to toggle the visibility of.
parent
A jQuery selector indicating what parent of the accordion trigger to force grouped mode.
.accordion("show")
Shows a collapsed accordion element.
.accordion("hide")
Hides a collapsed accordion element.
.accordion("toggle")
Toggles the visibility of a collapsed accordion element.

Events

The accordion exposes the following events allowing the developer to tap into its behaviour.

show.accordion.responsive
This event is fired immediately when the plugins show instance method is invoked.
shown.accordion.responsive
This event is fired when the plugins shown instance method is invoked once the collapsed region of the given instance has been made visible to the user.
hide.accordion.responsive
This event is fired immediately when the plugins hide instance method is invoked.
hidden.accordion.responsive
This event is fired once when the plugins hidden instance method is invoked and the collapsed region of the given instance has been hidden from the user.

Data API

The accordion's default behaviour is bound using a markup API that allows running the plugin without writing a single line of JavaScript. If you need to override this behaviour the following code will allow you to do so. 

// Override the default behaviour
$(document.body).off("click.accordion.responsive", ":attrStart(data-accordion)", function (event) {
    // Your custom behaviour...
}); 

// Bind to the show event.
$("YOUR_SELECTOR").on("show.accordion.responsive", function(event) {
    // Your custom behaviour...
});
                                               
// Bind to the shown event.
$("YOUR_SELECTOR").on("shown.accordion.responsive", function(event) {
    // Your custom behaviour...
});

// Bind to the hide event.
$("YOUR_SELECTOR").on("hide.accordion.responsive", function(event) {
    // Your custom behaviour...
});

// Bind to the hidden event.
$("YOUR_SELECTOR").on("hidden.accordion.responsive", function(event) {
    // Your custom behaviour...
});
